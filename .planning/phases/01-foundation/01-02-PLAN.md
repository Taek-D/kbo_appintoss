  <action>
    1. `src/app/api/auth/login/route.ts` — POST:
       - Request body: { authorizationCode: string, referrer: TossReferrer }
       - Zod 스키마로 body 검증
       - exchangeAuthCode(authorizationCode, referrer) → { accessToken }
       - getTossUserKey(accessToken) → { userKey }
       - upsertUser(userKey) → User
       - httpOnly cookie에 session_token 저장 (accessToken, secure: true, sameSite: 'lax', maxAge: 3600)
       - Response: 200 { success: true, user: { id, team_code, subscribed } }
       - 에러 시: 401 { error: string } (authCode 무효) 또는 500 { error: string }

    2. `src/app/api/auth/me/route.ts` — GET:
       - cookies()에서 session_token 읽기
       - session_token 없으면 → 401 { error: 'Unauthorized' }
       - getTossUserKey(session_token) → { userKey }
       - getUserByTossKey(userKey) → User | null
       - User가 null이면 → 404 { error: 'User not found' }
       - Response: 200 { user: { id, team_code, subscribed } }

    3. `src/app/api/subscription/route.ts`:
       - PUT: 응원팀 저장/변경
         - Request body: { team_code: TeamCode }
         - Zod로 team_code 검증 (유효한 TeamCode인지)
         - 세션에서 userId 추출 (me 엔드포인트와 동일한 로직)
         - updateTeamCode(userId, teamCode)
         - Response: 200 { success: true, user }
       - DELETE: 구독 해제
         - 세션에서 userId 추출
         - updateSubscription(userId, false)
         - Response: 200 { success: true }

    모든 Route Handler에서:
    - try/catch로 감싸서 에러 처리
    - 에러 시 { error: string } 형태 반환
    - 로거로 에러 기록 (console.log 금지)
    - NextRequest/NextResponse 사용 (App Router Route Handlers)
